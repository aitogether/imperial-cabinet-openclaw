# 治理规则 · Governance

## 任务状态机

内阁都察制定义了 7 个标准任务状态，确保任务流转有序可追踪：

| 状态 | 说明 | 负责角色 |
|------|------|---------|
| 待拆解 | 皇帝下旨，尚未拆解 | 御前首辅 |
| 已拆解 | 内阁完成任务拆解 | 内阁 |
| 审核中 | 提交御史台审核 | 御史台 |
| 已批准 | 审核通过，可执行 | 御史台 |
| 执行中 | 各部正在执行 | 八部 |
| 待验收 | 执行完成，等待验收 | 御前首辅 |
| 已完成 | 验收通过，任务结束 | 御前首辅 |
| 已取消 | 任务被取消 | 御前首辅 |
| 封驳 | 审核不通过，打回重做 | 御史台 |

### 合法状态转换

```
待拆解 ──→ 已拆解 ──→ 审核中 ──→ 已批准 ──→ 执行中 ──→ 待验收 ──→ 已完成
   │                      │                                          │
   └──→ 已取消            └──→ 封驳 ──→ (回到已拆解)            └──→ 执行中
                                                                    (打回重做)
```

转换白名单：

| 当前状态 | 可转为 |
|---------|--------|
| 待拆解 | 已拆解、已取消 |
| 已拆解 | 审核中 |
| 审核中 | 已批准、封驳 |
| 封驳 | 已拆解 |
| 已批准 | 执行中 |
| 执行中 | 待验收、已取消 |
| 待验收 | 已完成、执行中 |
| 已完成 | （终态） |
| 已取消 | （终态） |

### 快速通道（跳过审核）

以下场景可以跳过「审核中」状态，直接从「已拆解」进入「执行中」：

- 简单查询类任务（查日程、查消息）
- 用户明确说「不用审」的指令
- 紧急任务（用户标记加急）

## Agent 间通讯权限矩阵

核心原则：**用户只和御前首辅对话；八部之间不直接通讯。**

| From ↓ \ To → | 御前首辅 | 内阁 | 御史台 | 工部 | 户部 | 兵部 | 吏部 | 礼部 | 刑部 | 翰林院 | 东厂 |
|---------------|---------|------|--------|------|------|------|------|------|------|--------|------|
| **御前首辅** | — | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| **内阁** | ✅ | — | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | |
| **御史台** | ✅ | ✅ | — | | | | | | | | ✅ |
| **其他各部** | ✅ | | | — | | | | | | ✅ | |
| **东厂** | ✅ | | ✅ | | | | | | | | — |

**简化规则**：
1. 御前首辅可以给所有人发消息
2. 所有人可以给御前首辅发消息
3. 八部之间不直接通讯，必须通过御前首辅中转
4. 御史台可以给御前首辅、内阁、东厂发消息
5. 东厂只向御前首辅和御史台报告

## 御史台审核规则

### 必须审核的场景

- 内阁拆解的完整方案
- 工部的重大技术方案（涉及架构变更）
- 礼部的对外发布内容
- 兵部的安全策略方案
- 户部的重大预算决策
- 刑部的合规方案

### 审核标准

1. **完整性**: 是否有明确的输入、输出、步骤、验收标准
2. **风险**: 是否有数据丢失、安全、合规风险
3. **资源**: Token / 时间 / 人力估算是否合理
4. **可回滚**: 失败后是否有回退路径

### 封驳处理

御史台封驳时必须附详细理由，由御前首辅转内阁重新规划。同一任务最多封驳 2 次，第 3 次由御前首辅请示用户决定是否继续或取消。
