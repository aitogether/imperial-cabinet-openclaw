# 示例：一人公司内容流水线

## 场景描述

一人公司搭建日更内容流水线，覆盖从选题到发布的全流程。

## 流程概览

```
选题 → 资料收集 → 写稿 → 审核 → 发布 → 复盘
```

## 各部职责分工

### 御前首辅（调度中枢）

- 接收「开始今日内容生产」指令
- 将任务拆解并路由到各部门
- 汇总各部门产出，确认发布完成
- 向用户汇报：今日发布了什么、耗时多少、有什么问题

### 内阁（规划）

- 制定内容日历和选题策略
- 确定本周/本月的内容主题
- 设定每个选题的验收标准（字数、格式、目标读者）

### 吏部（流程）

- 设计内容生产的 SOP：
  1. 每日 9:00 自动触发选题
  2. 资料收集 30 分钟
  3. 写稿 60 分钟
  4. 审核 15 分钟
  5. 发布 15 分钟
- 维护任务流转规则

### 礼部（写作）

- 执行实际的写稿工作
- 保持风格一致性
- 生成配图描述（如需要）

### 工部（技术）

- 搭建自动化工具：
  - 热点监控脚本（抓取行业关键词）
  - 内容模板引擎
  - 自动发布到各平台的 API 集成
- 性能优化（减少等待时间）

### 御史台（审核）

- 审核每篇稿件的质量：
  - 事实准确性
  - 风格一致性
  - 是否有敏感内容
  - SEO 基本要素是否齐全
- 不合格则封驳打回

### 户部（成本）

- 统计每日内容生产的 Token 消耗
- 计算单篇内容成本
- 评估 ROI（阅读量 vs 生产成本）

### 兵部（安全）

- 管理各平台 API 密钥
- 确保发布账号安全
- 备份已发布内容

### 刑部（合规）

- 检查版权合规（图片、引用）
- 检查广告法合规（禁用词）
- 隐私合规（是否泄露个人信息）

### 翰林院（文档）

- 维护内容模板库
- 沉淀写作风格指南
- 记录复盘经验

### 东厂（监控）

- 记录每次内容生产的完整链路
- 统计各环节耗时
- 检测异常（如审核驳回率突然升高）
- 每周生成效率报告

## 一次典型执行

```
用户: "开始今日内容生产"

御前首辅: 收到。今日选题：「AI 自动化写作工具对比」。
  → 内阁确认选题和验收标准
  → 吏部走 SOP 流程
  → 礼部开始写稿（预计 60 分钟）
  → 稿件完成后 → 御史台审核
  → 审核通过 → 工部执行自动发布
  → 东厂记录全链路数据
  → 户部统计成本

御前首辅: 汇报：
  结论：今日文章已发布。
  证据：文章链接 / 审核记录 / 耗时统计。
  下一步：是否需要调整明日选题？
```
