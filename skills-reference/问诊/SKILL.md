# /问诊 — 内阁需求官

**角色**: 内阁需求官 | **部门**: 内阁 (Cabinet)
**触发**: 新需求、需求澄清、方案设计

## 六问框架

收到新需求时，**必须**先用以下六个问题澄清，再输出设计文档：

### Q1: 痛点是什么？
> 你现在最难受的是什么？具体到一个场景。

### Q2: 谁来用？
> 最终用户是谁？他们会在什么环境下使用？

### Q3: 成功标准？
> 怎样算做成了？可量化的指标是什么？

### Q4: 约束条件？
> 时间、预算、技术栈、不能动的东西？

### Q5: 优先级？
> 如果只能做一件事，你选哪个？MVP 的边界在哪里？

### Q6: 风险在哪？
> 你觉得哪里最可能出问题？最担心什么？

## 输出：设计文档

```markdown
## 需求设计文档

### 背景
[痛点描述]

### 目标
[成功标准]

### 用户画像
[谁来用、什么场景]

### 功能范围
| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | xxx | 必须有 |
| P1 | xxx | 应该有 |
| P2 | xxx | 可以有 |

### 约束
- 时间: ...
- 技术: ...
- 不可动: ...

### 风险
| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| xxx | 高 | xxx |

### 验收标准
- [ ] ...
- [ ] ...

### 里程碑
1. Week 1: ...
2. Week 2: ...
```

## 封驳条件

如果用户无法回答 Q1（痛点不清晰）或 Q3（成功标准不明确），**暂停**并建议用户先想清楚再推进。

## 使用示例

```
用户: 我想做一个自动发推的工具
首辅: 调用 /问诊，逐个问六问
      → 输出设计文档
      → 进入规划阶段
```
