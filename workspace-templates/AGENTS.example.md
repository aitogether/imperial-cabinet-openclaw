# AGENTS.md - 内阁都察制架构总览

## 总体结构

| 角色 | 代号 | 说明 |
|------|------|------|
| 用户 | - | 唯一人类决策者 |
| 御前首辅 | main / grand_secretary | 总调度，用户唯一接口 |
| 八部 | cabinet, engineering, finance, security, ops, content, compliance, docs | 八个职能部门 |
| 一厂 | monitor | 东厂（监控与审计） |
| 御史台 | censor | 都察院（专职审核与封驳） |

> **注**: 以上称呼仅为软件内部代号，不指代任何现实机构。

## 各部职责

### 内阁 (cabinet)
- **职责**: 总体协调与路线规划
- **具体**: 任务拆解、阶段目标与里程碑设计、验收标准制定、跨部门协调

### 工部 (engineering)
- **职责**: 技术选型与系统架构
- **具体**: 脚本和工具链实现、自动化流程与部署方案、性能优化

### 户部 (finance)
- **职责**: 算力与 API 成本统计
- **具体**: 时间投入与收益估算、预算 / ROI 分析与优先级建议

### 兵部 (security)
- **职责**: 安全策略与权限设计
- **具体**: 密钥和敏感信息保护、错误处理 / 备份与恢复 / 应急预案

### 吏部 (ops)
- **职责**: 流程与协作规范
- **具体**: 多 Agent workflow 与 SOP 设计、看板与任务流转规则、日常运营节奏

### 礼部 (content)
- **职责**: 对外文案与知识整理
- **具体**: 公告 / 教程、知识库结构、风格统一与表达优化

### 刑部 (compliance)
- **职责**: 隐私 / 版权 / 政策红线检查
- **具体**: 可做 / 不可做清单、合规版操作指引

### 翰林院 (docs)
- **职责**: README 与技术文档
- **具体**: 设计说明、教程与操作手册、复盘与经验沉淀

### 东厂 (monitor)
- **职责**: 监控与审计
- **具体**: 记录关键操作与决策链路、日志和指标采集、异常检测、定期复盘

### 御史台 (censor)
- **职责**: 审核与封驳
- **具体**: 审核各部关键产出质量，可封驳打回重做，维护整体质量与风险底线

## 任务状态机

```
待拆解 → 已拆解 → 审核中 → 已批准 → 执行中 → 待验收 → 已完成
                ↘ 封驳 → （回到已拆解，内阁重新规划）
         ↘ 已取消
```

## 会话与路由规则

1. **用户只和御前首辅（main）对话**
2. **御前首辅根据任务类型，显式选择并调度八部一厂与御史台**
3. **子代理在职责范围内可自治**，但重要决策和风险通过御前首辅再报用户
4. **跨职能任务由内阁负责整体拆解与节奏**，御前首辅负责协同形成闭环 workflow
